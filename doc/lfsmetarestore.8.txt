lfsmetarestore(8)
=================

== NAME

lfsmetarestore - replay LizardFS metadata change logs or dump LizardFS metadata image

== SYNOPSIS

[verse]
*lfsmetarestore* [*-z*] *-m* 'OLDMETADATAFILE' *-o* 'NEWMETADATAFILE' ['CHANGELOGFILE'...]

[verse]
*lfsmetarestore* *-m* 'METADATAFILE'

[verse]
*lfsmetarestore* [*-z*] *-a* [*-d* 'DIRECTORY']

[verse]
*lfsmetarestore -v*

[verse]
*lfsmetarestore -?*

== DESCRIPTION

When *lfsmetarestore* is called with both *-m* and *-o* options, it replays given 'CHANGELOGFILEs'
on 'OLDMETADATAFILE' and writes result to 'NEWMETADATAFILE'. Multiple change log files can be given.

*lfsmetarestore* with just *-m* 'METADATAFILE' option dumps LizardFS metadata image file in human
readable form.

*lfsmetarestore* called with -a option automatically performs all operations needed to merge change
log files. Master data directory can be specified using -d 'DIRECTORY' option.

*-v*::
print version information and exit

*-?*::
print version information and exit

*-a*::
autorestore mode (see above)

*-d* 'DATAPATH'::
master data directory (for autorestore mode)

*-m* 'METADATAFILE'::
specify input metadata image file

*-o* 'NEWMETADATAFILE'::
specify output metadata image file

*-z*::
ignore metadata checksum inconsistency while applying changelogs

== FILES

*metadata.lfs*::
Lizard File System metadata image as read by *lfsmaster* process

*metadata.lfs.back*::
Lizard File System metadata image as left by killed or crashed *lfsmaster* process

*changelog.***.lfs*::
Lizard File System metadata change logs
